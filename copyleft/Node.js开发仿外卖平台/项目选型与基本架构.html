# 项目选型与基本架构

# 项目选型

  • RESTful Api：REST风格的Api接口 - 关键字：REST、前后端分离、统一接口
  • Koa：基于Node.js的下一代Web框架

RESTful Api 原则：

  1. 尽量用名词复数作为资源
  • 获取用户列表：/users - https://api.github.com/users
  • 获取某个用户，主资源名就是用户：/users/{username} - https://api.github.com/users/huansheng1
  1. 资源标识与嵌套，获取某个资源下的子资源
  • 某个用户的某个仓库信息，所以仓库为主资源：/repos/huansheng1/Actions - https://api.github.com/repos/huansheng1/Actions
  1. 通过请求方式决定动作
  • 获取某个资源 - GET
  • 新建某个资源 - POST
  • 替换某个资源 - PUT
  • 修改某个资源 - PATCH
  • 删除某个资源 - DELETE
  • 对于部分操作，不好划分的，也可使用 POST+资源/操作词的方式，当然大部分情况不应这样做
  1. 常见返回错误码：2xx正确返回，3xx重定向，4xx客户端错误，5xx服务器错误

RESTful Api 设计我们推荐 Github 的设计，作为最出名的开原仓库，其 Api 设计可谓教科书级的，就是其中文文档翻译不全，还很怪，所以还是啃啃英文文档吧：Users - GitHub Docs (opens new window)

# 路由 koa-router

路由：通过区分请求地址、请求类型、请求参数来明白后端程序需要对应执行的不同任务

  1. 安装路由模块koa-router：npm install koa-router
  2. 写下测试代码：
const Koa = require('koa');
const app = new Koa();
const Router = require('koa-router');
const UserRouter = new Router({
    prefix: '/users'
});
UserRouter.get('/:id', async (ctx, next) => {
    // koa-router提供的/:id形式取接口地址params参数，原生koa是不支持的
    ctx.body = `你好，${ctx.params.id}！`;
    await next();
})
UserRouter.post('/', async (ctx, next) => {
    ctx.body = {
        code: 200,
        message: '创建用户成功'
    };
    await next();
})
app.use(UserRouter.routes());
// 如果不用这个方法，默认未实现的全部返回的是 404 Not Found
// 用了这个方法，会将常见未实现的请求方式返回 405 Method Not Allowed
// 不常用的请求方式则是返回 501 Not Implemented
app.use(UserRouter.allowedMethods());
// 注意不注释的话，后者会将前者返回结果覆盖
// app.use(ctx => {
//     ctx.body = '你好啊，饿了么';
// })
app.listen(3000);

# 断点调试与获取参数

一个请求的话会有以下几个参数：

  1. router params路由参数：/users/999，这需要koa-router模块的支持，然后我们在ctx.params取到
  2. query params请求参数：/users?q=name，我们在ctx.query取到
  3. headers协议头，我们可以在ctx.req.headers取到
  4. body主体参数，这需要koa-bodyparser的支持，然后我们在ctx.request.body取到

# 项目结构重构

  1. 重构router路由文件，新建一个app目录，将index.js作为入口文件：
const Koa = require('koa');
const app = new Koa();
// 引入解析body的模塊
const bodyparser = require('koa-bodyparser');
// 强烈注意bodyparser()，没有括号会报错TypeError: ctx.onerror is not a function
app.use(bodyparser());
// 引入router
const routerImport = require('./routers');
routerImport(app);
app.listen(3000);
app.on('error', err => {
    console.log('错误：', err);
});

module.exports = app => {

    fs.readdirSync(__dirname).forEach(filePath => {
        // 排除自身文件
        if (filePath === 'index.js') return;
        const currentRouter = require('./' + filePath);
        app.use(currentRouter.routes()).use(currentRouter.allowedMethods());
    })
}
